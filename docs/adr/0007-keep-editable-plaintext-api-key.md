# 保留可手工编辑的明文 API Key

DeepSeek API Key 继续保存在本机 JSON 配置文件中，以支持用户直接编辑和替换，不使用系统 Keychain 或 DPAPI 加密。作为最低保护，配置文件必须限制为当前系统用户可读写，设置页和日志不得输出完整 Key，清除配置时必须删除 Key 字段，并在维护文档中明确明文存储风险。

实现层面，POSIX 平台写入或重写本地配置文件后必须收紧到 `0o600`；Windows 继续依赖用户配置目录 ACL。运行日志、DeepSeek 失败响应和学生端公开状态需要统一脱敏，只允许显示“已配置 / 未配置”等状态，不得回显完整 Key。
