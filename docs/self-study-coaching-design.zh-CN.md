# 单机直连自学辅导设计

本文记录本地客户端自学辅导能力的已确认产品边界与待确认设计问题。它描述下一阶段目标，不代表所有能力已经实现。

## 产品边界

- 学生在个人电脑上独立使用，不建设账号、班级、教师端或云端教学记录
- 学生配置自己的 DeepSeek API Key，客户端按需直连 DeepSeek
- 请求不经过本项目自建服务器
- 当前版本不支持其他 AI 服务商或自定义接口地址
- 默认使用速度快、费用低的 DeepSeek Flash 模型
- 设置页允许家长在受支持的 Flash 与 Pro 模型之间切换
- 学生主界面不显示模型选项
- 代码内部保留 Provider 边界，但不作为当前产品能力暴露
- 辅导目标是帮助学生继续创作或解决当前编程问题，而不是代替学生完成作品

## 交互模式

### 主动求助模式

仅在学生主动请求帮助时，客户端发送当前作品摘要并生成新提示。

- 使用当前角色与作品摘要生成请求签名
- 请求签名与当前有效提示一致时，不重复调用 DeepSeek
- 重复点击继续显示现有推荐，不生成随机变化的新提示
- 作品发生有效变化或切换角色后，下一次点击才生成新提示

### 自动辅导模式

检测到有效作品变化后，客户端直接请求 AI 并刷新提示。为避免连续操作产生密集请求：

1. 首次保存有效 AI Key 后，默认启用自动辅导模式
2. 设置页明确说明自动辅导会使用用户配置的 Key 持续产生 API 调用，并允许切换为主动求助模式
3. 每次有效作品变化重新开始 3 秒静默计时
4. 学生按当前推荐顺序搭建积木时，保持现有提示，不发起新请求
5. 当前推荐序列全部完成后，等待 3 秒静默窗口再请求下一轮提示
6. 学生明显偏离当前推荐方向时，旧推荐立即失效，等待 3 秒静默窗口后重新分析
7. 同一时间只允许一个请求进行
8. 请求期间发生新变化时，只保留最新作品状态
9. 当前请求完成后，基于最新状态重新判断是否需要请求
10. 两次自动请求之间至少间隔 15 秒，间隔期间只追踪最新状态
11. 学生主动点击生成提示时可以立即请求，不受 15 秒自动请求间隔限制

## 辅导上下文

- 同一客户端运行期间、同一个 Scratch 作品构成一个辅导会话
- AI 至少可以参考上一轮提示及学生之后的作品变化
- 主界面始终只展示当前一轮提示，不提供历史推荐列表
- 旧推荐仅作为内部短期上下文，避免重复建议
- 切换作品或退出客户端时清除辅导上下文
- 辅导上下文仅保存在内存中，不写入本地持久化存储

## 提示深度

- 默认面向低年级学生，不使用只有方向、没有具体操作对象的轻提示
- 直接提供当前下一步真正需要的 1–3 个关键积木
- 最多显示 3 个，不使用本地建议强行补满
- 关键积木组成一个小型连接结构，不作为无序候选集合
- 结构需要表达前后连接、条件槽和内部嵌套关系
- 每个关键积木附一句简短中文原因
- 不展示额外文字示例
- 不一次性给出完整作品脚本或完整解题步骤

## 学生端输出

- 只显示一句简短总提示
- 显示一个由 1–3 个积木组成的推荐积木结构
- 每个积木只附一句简短原因
- 不显示追问或要求学生输入回答
- 不显示长篇分析、重复的下一步说明或完整教程
- 不显示完成勾选、进度、评分、星级或奖励动画
- 内部辅导诊断与技术错误不进入学生端输出

## 加载过渡

- 学生完成当前推荐并等待下一轮时，保留当前积木结构
- 保留期间显示“正在准备下一步”，不清空界面
- 学生切换角色或改变创作方向时，旧提示立即隐藏
- 旧提示失效后只显示简洁加载状态
- AI 请求失败时直接展示本地基础提示，不恢复已失效的旧提示

## 语音边界

- 第一版不提供自动语音朗读
- 提示更新时保持静音，不打断学生创作
- 第一版不引入系统 TTS 或第三方语音服务
- 未来可以单独评估可选的“朗读提示”按钮，但不作为当前范围

## 完成判定

- 推荐积木使用与 Scratch 脚本一致的结构表示
- 完成判定同时检查积木类型与连接关系
- 完成判定只用于内部触发下一轮提示，不作为学生端状态展示
- 前后积木需要按推荐顺序连接
- 条件积木需要出现在对应条件槽中
- 子堆栈积木需要放在对应控制积木内部
- 不要求数字、文字、变量名、角色目标或菜单选项与提示默认值一致
- 学生修改参数值仍视为完成，并保留其创作方向
- 积木仅散落在工作区或出现在无关脚本中，不算完成本轮推荐

## 积木有效性

- 推荐积木必须属于官方支持白名单
- 推荐积木必须存在于学生当前 Scratch 工具箱或已加载扩展中
- AI 返回未知、编造或当前不可用的积木时，直接丢弃该项
- 不把无效积木映射或替换为语义相近的其他积木
- 过滤后只保留剩余有效积木，不为补足数量加入新积木
- 如果过滤后没有任何有效积木，切换到本地基础提示
- 本地基础提示独立生成 1–3 个当前可用积木，不根据无效积木做近似替换
- 学生主界面只显示“已使用基础提示”，不展示过滤或解析错误

## AI 返回协议

- DeepSeek 只返回受约束 JSON，不返回 Blockly XML
- JSON 描述积木 opcode、前后连接、条件输入和子堆栈关系
- 客户端使用 schema 校验结构，并应用官方白名单与当前工具箱过滤
- 客户端根据验证后的结构自行生成 Blockly XML
- 不执行 AI 返回的代码、脚本或其他自由格式内容
- 协议验证失败时按失败降级规则使用本地基础提示

## 作品驱动

- 学生不需要输入创作目标或自然语言问题
- 主界面不提供聊天窗口或文字求助输入框
- 学生可以自由决定想做什么，客户端只分析已经完成的作品
- AI 根据当前作品判断最合适的下一小步，并返回有序关键积木
- 代码中的可选目标参数不作为面向学生的产品入口

## 角色切换

- 推荐积木结构只针对当前选中的角色
- 学生切换角色时，立即结束旧角色的提示稳定期并隐藏旧提示
- 新角色基于自身完整脚本和全作品摘要重新生成提示
- 学生切回原角色时按最新作品重新分析，不恢复旧提示
- 角色切换不会结束整个辅导会话，也不会清空全局短期上下文

## 空白作品

- 完全空白的作品不触发 AI 请求
- 不预设“绿旗、移动、说话”等通用起步积木
- 主界面只显示“先开始你的创作，搭出第一小步后我再来帮你”
- 检测到第一个有效积木后，才进入主动求助或自动辅导流程

## 适应学生方向

- 每轮推荐积木只是建议，不是必须完成的任务
- 学生未采用旧建议时，不显示“做错了”或“尚未完成”
- 学生按推荐方向推进时，当前提示保持稳定
- 学生完成整组推荐后，生成下一轮提示
- 学生转向其他创作方向时，旧推荐立即失效
- 下一轮完全基于最新作品重新判断创作方向
- 不要求学生删除新内容或回退到旧建议

## 内部诊断

- 可以保留内部问题诊断，用于选择更合适的下一步积木
- 学生主界面不展示独立的错误、警告或问题列表
- 不对学生作品评分，也不标记为正确或错误
- 必要的诊断信息转写为积木旁的一句温和原因
- 技术诊断字段不作为学生端产品概念

## 只读边界

- 推荐积木只使用 Scratch 原版样式展示
- 客户端不向 Scratch 自动插入、拖放或连接积木
- 客户端不修改变量、角色、造型、声音或项目设置
- 学生需要自己在 Scratch 中找到并按提示顺序搭建积木
- 客户端观察学生完成后的作品变化，再生成下一轮提示

## 作品摘要边界

- 当前角色发送完整脚本结构
- 其他角色仅发送少量脚本摘要，以保留广播、碰撞和共享变量等跨角色关系
- 可以发送变量名称与当前值、已用模块、已加载扩展和检测到的编程概念
- 不发送造型、声音、图片等媒体资源
- 不发送完整 `.sb3` 或原始项目 JSON

## 失败降级

- 未配置 AI Key 时，继续使用本地基础提示
- AI 请求超时、断网、额度不足或返回异常时，自动回退到本地基础提示
- 学生主界面只显示“AI 暂时不可用，已使用基础提示”
- HTTP 状态码、服务端响应和异常堆栈不展示给学生
- 详细错误仅写入设置页诊断信息或本地日志

## 家长提示

- 保存 AI Key 时显示：“自动辅导会在作品变化后使用此 Key 请求 AI，可能产生 API 费用。”
- 默认选择自动辅导模式，同时提供主动求助模式选项
- 费用与 Key 配置说明放在设置页，不占用学生创作主界面

## Key 存储

- DeepSeek API Key 继续明文保存在本机 JSON 配置文件
- 正式支持用户直接编辑配置文件替换 Key
- 配置文件必须限制为当前系统用户可读写
- 设置页只显示是否已配置，不回显完整 Key
- 日志、错误信息和诊断数据不得包含完整 Key
- 清除 Key 时删除配置文件中的对应字段
- 维护文档明确说明明文存储的安全风险
