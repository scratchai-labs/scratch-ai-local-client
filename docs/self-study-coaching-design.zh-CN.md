# 单机直连自学辅导设计

本文记录本地客户端自学辅导能力的已确认产品边界与待确认设计问题。它描述下一阶段目标，不代表所有能力已经实现。

## 产品边界

- 学生在个人电脑上独立使用，不建设账号、班级、教师端或云端教学记录
- 学生配置自己的 AI API Key，客户端按需直连第三方 AI 服务商
- 请求不经过本项目自建服务器
- 辅导目标是帮助学生继续创作或解决当前编程问题，而不是代替学生完成作品

## 交互模式

### 主动求助模式

仅在学生主动请求帮助时，客户端发送当前作品摘要并生成新提示。

### 自动辅导模式

检测到有效作品变化后，客户端直接请求 AI 并刷新提示。为避免连续操作产生密集请求：

1. 首次保存有效 AI Key 后，默认启用自动辅导模式
2. 设置页明确说明自动辅导会使用用户配置的 Key 持续产生 API 调用，并允许切换为主动求助模式
3. 每次有效作品变化重新开始 3 秒静默计时
4. 连续 3 秒没有新的有效作品变化时发起请求
5. 同一时间只允许一个请求进行
6. 请求期间发生新变化时，只保留最新作品状态
7. 当前请求完成后，基于最新状态重新判断是否需要请求
8. 两次自动请求之间至少间隔 15 秒，间隔期间只追踪最新状态
9. 学生主动点击生成提示时可以立即请求，不受 15 秒自动请求间隔限制

## 辅导上下文

- 同一客户端运行期间、同一个 Scratch 作品构成一个辅导会话
- AI 至少可以参考上一轮提示及学生之后的作品变化
- 切换作品或退出客户端时清除辅导上下文
- 辅导上下文仅保存在内存中，不写入本地持久化存储

## 提示深度

- 默认面向低年级学生，不使用只有方向、没有具体操作对象的轻提示
- 直接提供当前下一步所需的约 3 个关键积木
- 关键积木按照建议使用顺序排列，不作为无序候选集合
- 每个关键积木附一句简短中文原因
- 不展示额外文字示例
- 不一次性给出完整作品脚本或完整解题步骤

## 作品驱动

- 学生不需要输入创作目标或自然语言问题
- 主界面不提供聊天窗口或文字求助输入框
- 学生可以自由决定想做什么，客户端只分析已经完成的作品
- AI 根据当前作品判断最合适的下一小步，并返回有序关键积木
- 代码中的可选目标参数不作为面向学生的产品入口

## 只读边界

- 推荐积木只使用 Scratch 原版样式展示
- 客户端不向 Scratch 自动插入、拖放或连接积木
- 客户端不修改变量、角色、造型、声音或项目设置
- 学生需要自己在 Scratch 中找到并按提示顺序搭建积木
- 客户端观察学生完成后的作品变化，再生成下一轮提示

## 作品摘要边界

- 当前角色发送完整脚本结构
- 其他角色仅发送少量脚本摘要，以保留广播、碰撞和共享变量等跨角色关系
- 可以发送变量名称与当前值、已用模块、已加载扩展和检测到的编程概念
- 不发送造型、声音、图片等媒体资源
- 不发送完整 `.sb3` 或原始项目 JSON

## 失败降级

- 未配置 AI Key 时，继续使用本地基础提示
- AI 请求超时、断网、额度不足或返回异常时，自动回退到本地基础提示
- 学生主界面只显示“AI 暂时不可用，已使用基础提示”
- HTTP 状态码、服务端响应和异常堆栈不展示给学生
- 详细错误仅写入设置页诊断信息或本地日志

## 家长提示

- 保存 AI Key 时显示：“自动辅导会在作品变化后使用此 Key 请求 AI，可能产生 API 费用。”
- 默认选择自动辅导模式，同时提供主动求助模式选项
- 费用与 Key 配置说明放在设置页，不占用学生创作主界面
