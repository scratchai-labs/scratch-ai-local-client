## 1. 这是什么

`Scratch AI 教练`（桌面伴随程序，Scratch Desktop Companion）是面向 **Scratch Desktop** 的本机 AI 伴随工具。

它**不会修改** Scratch 官方源码，也不会往作品里自动插入积木。  
它会：

1. 由伴随程序**受控启动**本机 Scratch Desktop  
2. 通过只读桥接读取当前作品与当前角色脚本  
3. 用 Scratch 原版风格展示「当前角色程序」与「推荐积木」  
4. 基于当前作品生成 **下一步关键积木提示**（默认走 DeepSeek；无 Key 或失败时回退本地基础提示）

一句话：**学生继续在 Scratch 里自己搭积木；教练窗口只观察、提示、推荐。**

---

## 2. 你能得到什么

| 能力 | 说明 |
| --- | --- |
| 连接本机 Scratch | 自动识别常见安装路径，也可手动选择 |
| 当前角色同步 | 显示正在编辑的角色 / 舞台 |
| 当前角色程序 | 以 Scratch 原版彩色积木只读渲染当前脚本 |
| 下一步提示 | 一句简短中文建议 + 1–3 个关键推荐积木结构 |
| 推荐积木 | 按官方积木样式展示；非法/不支持积木会被过滤 |
| 自动 / 手动刷新 | 可在设置里切换：积木变化自动刷新，或手动点击生成 |
| 本地回退 | 未配置 DeepSeek Key 或上游失败时，仍有本地基础提示 |
| 系统托盘 | 关闭窗口不退出，缩回托盘继续驻留 |

## 3. 系统要求

### 3.1 操作系统

- **Windows 10 / 11**（推荐 64 位）
- **macOS**（支持 `.zip` 便携包与 `.dmg` 安装包）

### 3.2 必备软件

- 已安装 **Scratch Desktop / Scratch 3 Desktop**
- 可选：本机 DeepSeek API Key（用于线上 AI 提示；没有也能用本地基础提示）

### 3.3 重要约束（请先读）

- 当前主流程是：**伴随程序启动 Scratch**，再建立只读连接。  
- **不支持**先手工双击打开 Scratch，再让伴随程序“附着”上去。  
- 如果 Scratch 已由用户手工打开：请先**完全退出 Scratch**，再回到伴随程序点「打开已选 Scratch」。

## 5. 安装与首次启动

### 5.1 Windows · 安装版（setup.exe）

1. 下载 `ScratchDesktopCompanion-setup.exe`  
2. 双击，按向导安装到固定目录（例如 `C:\ScratchDesktopCompanion\`）  
3. 启动后确认**系统托盘**出现伴随程序图标  
4. 打开主窗口，继续第 6 节首次连接流程

### 5.2 Windows · 便携版（portable.exe）

1. 将 `ScratchDesktopCompanion-portable.exe` 放到固定目录  
2. 双击运行  
3. 后续与安装版相同

> 提示：便携版启动通常比已解压目录稍慢；现场批量部署也可使用解压后的 `win-unpacked` 目录（若你从开发产物获得）。

### 5.3 macOS · DMG

1. 下载并打开 `ScratchDesktopCompanion-mac.dmg`  
2. 将 `ScratchDesktopCompanion.app` 拖到「应用程序」或自选目录  
3. 首次打开若被 macOS 安全策略拦截：在「系统设置 → 隐私与安全性」中允许打开  
4. 启动后确认菜单栏 / 托盘区域有伴随程序入口

### 5.4 macOS · ZIP 便携包

1. 解压 `ScratchDesktopCompanion-mac.zip`  
2. 得到 `ScratchDesktopCompanion.app`  
3. 双击运行（同样可能需要手动允许来自未签名开发者的应用）

> 当前正式发布链路**不做** macOS 签名 / 公证，首次打开可能需要额外信任步骤。

## 6. 五分钟上手（推荐流程）

按顺序做，成功率最高：

```text
启动伴随程序
  → 确认 / 选择 Scratch 软件路径
  → 点击「打开已选 Scratch」（不要自己先开 Scratch）
  → 等待状态变为「已连接」
  → （可选，详见第 8 节）打开 DeepSeek 设置添加 API Key
  → 在 Scratch 中创作；查看当前角色程序与下一步提示
```

### 6.1 选择 Scratch 软件

1. 打开伴随程序主窗口  
2. 若已自动识别，会显示「已选 Scratch」路径  
3. 若未识别，点 **「选择 Scratch 软件」**

可选路径示例：

**Windows**

- `Scratch.exe` / `Scratch 3.exe`  
- 常见安装目录下的可执行文件  
- 有效的桌面快捷方式 `.lnk`（优先选真实 exe）

**macOS**

- `/Applications` 或 `~/Applications` 下的  
  `Scratch.app` / `Scratch Desktop.app` / `Scratch 3.app`  
- 或应用包内真实可执行文件

### 6.2 打开已选 Scratch

1. 点击 **「打开已选 Scratch」**  
2. 等待约 3–10 秒  
3. 状态应变为类似 **「已连接」** / 「已连接到 Scratch Desktop」  
4. 主界面应能看到：
   - 已选 Scratch 路径  
   - 当前角色  
   - 同步时间  
   - 当前角色程序（原版积木视图）

### 6.3 开始获得提示

- **默认：自动刷新**  
  连接成功后，以及后续积木发生有效变化并稳定后，会自动请求下一步提示。  
- **也可：手动点击**  
  到「DeepSeek 设置」切到「手动点击」，再点主界面 **「生成下一步提示」**。

建议低年级课堂优先用**手动点击**，避免学生改积木时提示频繁跳动。

## 7. 主界面说明

主窗口大致分为：

### 7.1 顶部标题栏

- 产品名称与简短说明  
- **DeepSeek 设置** 按钮（打开独立设置窗口）

### 7.2 操作区按钮

| 按钮 | 作用 |
| --- | --- |
| 选择 Scratch 软件 | 手动指定 Scratch 可执行文件 / 应用 |
| 打开已选 Scratch | 由伴随程序受控启动 Scratch 并建立连接 |
| 重新连接 | 连接异常时重试桥接 |
| 生成下一步提示 | 主动请求一轮 AI / 本地提示 |

### 7.3 状态信息

- **状态**：是否已连接  
- **当前角色**：Scratch 中正在编辑的角色 / 舞台  
- **同步时间**：最近一次状态刷新时间  
- **已选 Scratch**：当前使用的可执行文件路径  
- **AI 配置摘要 / 提示来源**：是否已配置 Key、当前走 DeepSeek 还是本地基础提示

### 7.4 当前角色程序

- 标题会体现当前角色（例如「Cat 的程序」）  
- 以 Scratch 原版风格只读显示当前角色脚本  
- 多脚本时可在区域内滚动浏览  
- **只读**：不能在此窗口改积木；请回到 Scratch 操作

### 7.5 下一步提示 / 推荐积木

- 一句简短总提示（中文）  
- 1–3 个关键推荐积木（含简短原因）  
- 推荐结构可能包含前后连接、条件槽、子堆栈关系  
- 若作品已被判定为完整，可能只说明玩法/使用方式，不再强制塞推荐积木  
- 无 Key / 上游失败时，显示本地基础提示，保证学生仍可继续

## 8. 如何添加 DeepSeek API Key（重点）

正式安装包**不预置** API Key。若要使用线上 AI 生成更好的下一步提示，需要你自己在本机填写 DeepSeek Key。

### 8.1 准备：先拿到 Key

1. 打开 DeepSeek 开放平台（官方控制台，常见入口为 `https://platform.deepseek.com`）
2. 注册 / 登录账号
3. 进入 **API Keys**（API 密钥）页面
4. 新建一把密钥，**立刻复制并妥善保存**
5. 确认账号有可用余额（没余额时，Key 可能测得通但仍无法正常出提示）

> 安全提醒：  
> - 不要把 Key 发给学生、不要截图发群  
> - 不要把 Key 写进共享 U 盘上的公开文档  
> - 机房如需共用，建议由老师在每台电脑本地分别配置，或使用可轮换的专用 Key

### 8.2 在伴随程序中添加 Key（推荐步骤）

1. 启动 **Scratch AI 教练**（桌面伴随程序）
2. 在主窗口右上角点击 **「DeepSeek 设置」**  
   会打开独立窗口，标题为「DeepSeek 本地设置」
3. 在 **「DeepSeek API Key」** 输入框中粘贴你的 Key  
   （输入框是密码样式，不会明文一直显示）
4. 先点 **「测试 Key」**（强烈建议）  
   - 成功：会提示 Key 可用，并可能显示余额摘要  
   - 失败：常见为 Key 无效、网络不通、超时、余额不足
5. 测试通过后，点 **「保存 Key」**
6. 关闭设置窗口，回到主界面  
   主界面「AI 配置摘要 / 提示来源」应变为类似：  
   **「当前使用本机保存的 DeepSeek Key」** 或 **「当前提示来源：DeepSeek（模型名）」**
7. 连接 Scratch 后，点 **「生成下一步提示」** 验证是否真的走线上 AI

### 8.3 按钮分别做什么

| 按钮 | 作用 |
| --- | --- |
| 测试 Key | 用当前输入框内容（或已保存 Key）请求 DeepSeek 校验接口，检查是否可用 |
| 保存 Key | 把 Key 写入本机用户配置；之后生成提示会优先走 DeepSeek |
| 清除 Key | 删除本机已保存 Key；之后不再请求 DeepSeek，只使用本地基础提示 |

### 8.4 添加成功后的默认行为

- Key 以**明文**保存在当前电脑，并限制为**当前系统用户**可读写  
- **保存 Key 后，客户端默认会切到「手动点击」**  
  这样避免学生改积木时频繁自动请求、产生额外费用  
- 如需连续自动跟随，可再到设置里改回「自动刷新」并保存

### 8.5 模型选择（可选，但建议确认）

在同一设置窗口的 **「模型选择」**：

| 模型 | 说明 | 建议 |
| --- | --- | --- |
| `deepseek-v4-flash` | 默认；更快，更适合课堂连续请求 | 日常首选 |
| `deepseek-v4-pro` | 更强；可能更慢、费用更高 | 需要更稳判断时再用 |

选择后点 **「保存模型」**。  
主界面的「提示来源」会显示当前模型。

### 8.6 下一步提示触发方式（可选）

| 模式 | 行为 | 适用场景 |
| --- | --- | --- |
| 自动刷新 | 连接成功后、积木有效变化稳定后自动请求 | 自由探索、希望持续跟随 |
| 手动点击 | 仅在点击「生成下一步提示」时请求 | 课堂演示、控制费用、避免提示乱跳 |

改完后点 **「保存触发方式」**。

### 8.7 怎么确认 Key 已经生效

按下面清单自检：

1. 设置页状态不再提示「还没有保存本机 DeepSeek Key」
2. 主界面 AI 摘要显示已使用本机保存的 Key
3. 生成提示后，提示来源显示 **DeepSeek**，而不是「本地基础提示」
4. 若仍显示本地基础提示：重新打开设置 → 再测 Key → 再保存 → 再生成一次

### 8.8 更换 / 删除 Key

**更换：**

1. 打开「DeepSeek 设置」
2. 输入新 Key
3. 点「测试 Key」
4. 点「保存 Key」

**删除：**

1. 打开「DeepSeek 设置」
2. 点「清除 Key」
3. 之后只会使用本地基础提示

### 8.9 没有 Key 能不能用？

可以。  
未配置 Key、Key 无效、余额不足或上游失败时，会回退到**本地基础提示**，仍给出有限的下一步积木建议。  
线上 AI 质量通常更好，但不是硬性门槛。

### 8.10 添加 Key 常见问题

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 点了保存，但仍像没 Key | 只输入了没点「保存 Key」 | 重新粘贴后明确点保存 |
| 测试 Key 失败 | 复制少了字符 / 混入空格 / Key 已作废 | 回平台重新创建并完整复制 |
| 测试成功但提示仍走本地 | 网络中断、模型返回无法校验、请求超时 | 看主界面提示来源；重试「生成下一步提示」；必要时重启伴随程序 |
| 余额不足 | 平台账号欠费 | 充值后再测 Key |
| 机房每台都要配？ | Key 存在本机用户配置里，不随安装包分发 | 每台电脑各自配置，或由老师批量配置 |

### 8.11 老师批量配置建议（机房）

1. 先用一台样机验证：填 Key → 测试 → 保存 → 生成提示成功  
2. 再在其余机器重复同样步骤  
3. 课堂建议统一设为 **手动点击**，避免同时自动刷提示  
4. 课程结束后如需回收权限，可在各机点「清除 Key」，并在 DeepSeek 平台轮换/作废旧密钥

## 9. 学生日常用法

1. **先开伴随程序**，不要先双击 Scratch  
2. 确认已选 Scratch 路径正确  
3. 点 **「打开已选 Scratch」**  
4. 在 Scratch 里正常创作（拖积木、换角色、打开 `.sb3` 均可）  
5. 看伴随窗口：
   - 当前角色是否一致  
   - 当前角色程序是否更新  
   - 下一步提示 / 推荐积木  
6. 需要时点 **「生成下一步提示」**（手动模式）  
7. 关掉伴随主窗口 → 程序会缩回托盘，不一定退出  
8. 下课退出：从托盘菜单选择退出（或结束应用）

**请记住：**

- 推荐积木是「建议」，不是标准答案  
- 学生必须自己在 Scratch 工具箱里找到并搭建  
- 换角色后，旧角色的提示会失效，以当前角色 / 最新作品为准  
- 空白作品通常不会触发有意义的 AI 辅导起点

## 10. 教师 / 机房建议

### 10.1 课前准备

- 确认每台机器已安装 Scratch Desktop  
- 预装伴随程序（安装版或固定目录便携版）  
- 首次开机后完成一次「选择 Scratch → 打开已选 Scratch」  
- 若使用线上 AI：提前配置并测试 DeepSeek Key  
- 抽查至少 3 台机器：连接、角色切换、生成提示、推荐积木显示

### 10.2 课中协作

- 状态异常时：先「重新连接」  
- 学生误开了 Scratch：让其完全退出 Scratch，再由伴随程序打开  
- 托盘图标可快速唤回主窗口  
- 建议课堂默认 **手动点击**，由老师控制何时请求提示

### 10.3 隐私与数据边界（简要）

- 不修改 Scratch 源码，不向作品写入积木  
- AI 请求发送的是当前作品的**结构化摘要**（脚本连接、字段、条件、变量/广播关系等），不是完整媒体资源包  
- 辅导上下文主要在当前运行会话内存中，不作为长期学生档案  
- API Key 只保存在本机用户配置中

## 11. 常见问题 FAQ

### Q1：状态一直停在「请先选择 Scratch 软件」？

- Scratch 是否真的安装了  
- 是否选中了正确的 `Scratch.exe` / `Scratch 3.exe` / `.app`  
- 桌面快捷方式是否失效  
- 重新点一次「选择 Scratch 软件」，优先真实可执行文件

### Q2：一直提示「请从伴随程序打开已选 Scratch」？

说明还没走通受控启动链路。  

处理：

1. 不要手工双击 Scratch  
2. 若 Scratch 已打开，先完全退出  
3. 回到伴随程序点「打开已选 Scratch」

### Q3：点了打开，但连不上？

- 看 Scratch 是否被启动  
- 点「重新连接」  
- 确认选中的是 Scratch 本体，不是错误快捷方式  
- 仍失败时收集日志（见第 12 节）

### Q4：当前角色程序是空白 / 不更新？

- 确认状态已是「已连接」  
- 在 Scratch 中确认该角色是否真有脚本  
- 切换角色或增删积木后再观察  
- 点「重新连接」  
- 若刚删光脚本，旧视图应被清空；若仍残留，重启伴随程序

### Q5：推荐积木区为空？

可能原因：

- 作品被判断为已较完整，仅返回使用说明  
- DeepSeek 返回了不可渲染/不在白名单的积木，已被过滤  
- 尚未配置 Key，且本地基础提示也未能给出结构  
- 请求仍在进行中

可尝试：再点「生成下一步提示」、检查 Key、查看提示来源文案。

### Q6：为什么显示「本地基础提示」而不是 DeepSeek？

常见原因：

- 未保存 API Key  
- Key 无效 / 余额不足  
- 网络或上游接口失败  
- 模型返回结构无法通过客户端校验，已安全降级

到设置页用「测试 Key」排查。

### Q7：macOS 打不开应用？

未签名应用会被 Gatekeeper 拦截。到「系统设置 → 隐私与安全性」允许打开，或按本机安全策略由管理员批准。

### Q8：关闭窗口后程序还在吗？

是。关闭主窗口默认**隐藏到托盘**，不会直接退出。需要彻底退出请用托盘菜单。

### Q9：可以附着到已经打开的 Scratch 吗？

**当前不支持。** 必须由伴随程序启动 Scratch。

### Q10：支持 Linux / 网页版 Scratch 吗？

当前主线只维护 **Windows + macOS 桌面本地版**，面向 Scratch Desktop。

## 14. 当前限制（使用前知情）

1. 只支持「伴随程序受控启动 + CDP 注入」，不支持附着手工打开的 Scratch  
2. 正式分发包无预置 Key；线上 AI 需自备 DeepSeek Key  
3. macOS 当前无自动签名 / 公证流程  
4. 推荐积木受官方 opcode 白名单与结构校验约束，模型胡编积木不会原样展示  
5. 界面默认面向中文用户；Scratch 语言会在桥接成功后尽量沿用上次记录的界面语言  
6. 本仓不包含服务器端；跨仓规划见 `scratch-ai-docs`
